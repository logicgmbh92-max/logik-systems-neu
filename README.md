## 🚀 Quick Start

### Prerequisites
- Python 3.9+
- Stripe account (for payments)
- OpenAI API key (for assessments)
- Supabase account (for database)

### Installation

```bash
# Clone repository
git clone https://github.com/logik-systems/compliance-platform.git
cd compliance-platform

# Install dependencies
pip install -r requirements.txt

# Create .env.local (copy from .env.example)
cp .env.example .env.local

# Add your API keys to .env.local
# STRIPE_API_KEY=sk_live_...
# OPENAI_API_KEY=sk-...
# SUPABASE_URL=https://...
# SUPABASE_SERVICE_ROLE_KEY=...

# Run locally
python main.py
```

### Test the API

```bash
# Health check
curl http://localhost:8000/

# Get disclaimer
curl http://localhost:8000/compliance/disclaimer

# Start an audit (example)
curl -X POST http://localhost:8000/v1/audit \
  -H "Content-Type: application/json" \
  -d '{
    "system_name": "Customer Recommendation Engine",
    "system_description": "ML model that recommends products based on user behavior",
    "ai_model_type": "Recommendation Engine",
    "use_case": "E-commerce personalization",
    "data_categories": ["user_behavior", "purchase_history", "demographics"],
    "tier": "audit_scan"
  }'
```

## 🔧 Deployment

### Vercel (Recommended)

1. **Connect GitHub Repository**
   - Go to vercel.com
   - Click "New Project"
   - Select your GitHub repository
   - Click "Import"

2. **Configure Environment Variables**
   - Go to Settings > Environment Variables
   - Add all variables from `.env.example`:
     - `STRIPE_API_KEY`
     - `OPENAI_API_KEY`
     - `STRIPE_WEBHOOK_SECRET`
     - `SUPABASE_URL`
     - `SUPABASE_SERVICE_ROLE_KEY`

3. **Deploy**
   - Click "Deploy"
   - Your API will be live at `https://your-project.vercel.app`

### Supabase Database Setup

Run these SQL commands in your Supabase SQL editor:

```sql
-- Create audits table
CREATE TABLE audits (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  audit_id TEXT UNIQUE NOT NULL,
  system_name TEXT NOT NULL,
  tier TEXT NOT NULL,
  status TEXT DEFAULT 'completed',
  assessment TEXT,
  disclaimer TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Create outreach_log table
CREATE TABLE outreach_log (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  outreach_id TEXT UNIQUE NOT NULL,
  prospect_name TEXT NOT NULL,
  prospect_company TEXT NOT NULL,
  prospect_email TEXT NOT NULL,
  prospect_role TEXT,
  use_case TEXT,
  message TEXT,
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT NOW()
);

-- Create payments table
CREATE TABLE payments (
  id BIGINT PRIMARY KEY GENERATED ALWAYS AS IDENTITY,
  stripe_invoice_id TEXT UNIQUE NOT NULL,
  amount_eur DECIMAL(10, 2),
  status TEXT DEFAULT 'completed',
  created_at TIMESTAMP DEFAULT NOW()
);

-- Create indexes for performance
CREATE INDEX idx_audits_audit_id ON audits(audit_id);
CREATE INDEX idx_outreach_email ON outreach_log(prospect_email);
CREATE INDEX idx_payments_invoice ON payments(stripe_invoice_id);
```

```
GET /
```
Returns service status and version.

```
GET /compliance/disclaimer
```

### Start Compliance Audit
```
POST /v1/audit
Content-Type: application/json

{
  "system_name": "string",
  "system_description": "string",
  "ai_model_type": "string",
  "use_case": "string",
```

Returns:
- Unique audit ID

- Compliance disclaimer
- Next steps for professional review

```
POST /webhook
```
Handles Stripe payment events. Configure in Stripe Dashboard:
- Endpoint URL: `https://your-domain.com/webhook`
- Events: `invoice.payment_succeeded`

## 🔐 Security Best Practices

### Environment Variables
- ✅ Store all secrets in platform environment variables
- ✅ Use `.env.local` for local development only
- ✅ Add `.env` and `.env.local` to `.gitignore`
- ❌ Never commit secrets to version control

### API Security
- ✅ All endpoints validate input
- ✅ CORS restricted to logik-systems.com
- ✅ Stripe webhooks verified with signature
- ✅ Rate limiting recommended (add to production)

### Data Protection
- ✅ All assessments stored in Supabase with encryption
- ✅ GDPR-compliant data handling
- ✅ Audit logs for compliance tracking
- ✅ Opt-out mechanism for outreach
